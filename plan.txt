# XtrazCon IT ERP — All Diagrams in Text/Tree Format

---

## 1. USER ROLE HIERARCHY (who reports to whom, top → bottom)

```
Super Admin (SADMIN)
│   Full system control — sees everything, creates roles/teams
│
└── Admin (ADMIN)
    │   Manages departments, templates, teams, reports
    │
    └── Manager (MANAGER)
        │   Oversees one department
        │
        └── Team Leader (TL)
            │   Leads a team, manages groups under it
            │
            └── Coordinator (COORD)
                │   Oversees one group, assigns work
                │
                └── Member / Employee (MEMBER)
                    │   Handles own tasks and leads
                    │
                    └── Intern (INTERN)
                            Task-only, no financial/HR access


      ── EXTERNAL USERS (outside the chain, scoped access only) ──

Client (CLIENT)              Vendor / Freelancer (VENDOR)
  → sees own projects,         → sees only assigned
    invoices, tickets            projects/tasks/files
```

**Rule:** each level can see its own level + everything below it. Client and Vendor never inherit anything — they only see what's explicitly assigned to them.

## 2. ORGANIZATIONAL CONTAINMENT (Department → Team → Group → User)

```
Department (e.g. SEO)
│
├── Team A                          (owned by: Rahul, role: Manager/TL)
│   │
│   ├── Group A1
│   │   ├── Amit      — Coordinator — Follow-up monitoring
│   │   ├── Karan      — Member      — SEO submissions
│   │   └── Priya      — Member      — Link building
│   │
│   ├── Group A2
│   │   └── (members...)
│   │
│   └── Group A3
│       └── (members...)
│
└── Team B
    └── (groups, members...)
```

Other departments follow the same pattern:
```
Sales | SEO | Development | Testing | Support | HR | Accounts | Management
   each → Team(s) → Group(s) → User(s)
```

## 4. ACCESS / VISIBILITY FLOW (what each role can see)

```
Own tasks/leads
   ▲
   │  (Member sees only this)
   │
Group tasks/leads
   ▲
   │  (Coordinator adds this)
   │
Team tasks/leads
   ▲
   │  (TL adds this)
   │
Department data
   ▲
   │  (Manager adds this)
   │
Everything (all departments, all data)
   ▲
   │  (Super Admin — full access)
```

Text form of the escalation:
```
Member      → own tasks, own leads
Coordinator → + group leads, group tasks
TL          → + team leads, team tasks
Manager     → + department data
Admin       → + assigned departments, template/team management
Super Admin → + everything, company-wide
```

## 7. DASHBOARD HIERARCHY (what renders per role)

```
Super Admin Dashboard
   → entire company, all departments, finance, KPIs, sales, HR

   Department Admin Dashboard
      → own department stats, team performance, attendance, tasks

      TL Dashboard
         → own groups, own team performance, pending work

         Member Dashboard
            → assigned work, attendance, meetings, KPIs
```

External:
```
Client Dashboard   → own projects, invoices, tickets, subscription only
Vendor Dashboard   → assigned projects, tasks, uploaded files only
```

## 9. FIREBASE SECURITY RULE LOGIC (text form)

```
IF role == "TL"        → allow read/write ONLY where team_id == user.team_id
IF role == "MANAGER"   → allow read/write ONLY where dept_id == user.dept_id
IF role == "SADMIN"    → allow read/write on everything
Never trust frontend-only permission checks — enforce in Firestore rules.
`
